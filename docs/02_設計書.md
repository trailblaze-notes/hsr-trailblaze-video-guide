# 設計書 — ゲーム進行度別 公式動画ガイド

作成日: 2026-07-18 / 作成: Claude（AI 駆動開発）

## 1. 技術選定

| 領域 | 採用 | 理由 | 捨てた選択肢と理由 |
|---|---|---|---|
| 言語 | TypeScript | データ構造（ゲーム/動画）の型安全。エディタ補完でデータ追加ミスを防ぐ | JS のみ（型がないとデータ追加時の事故が多い） |
| フレームワーク | Astro | コンテンツ主体サイトに最適。デフォルト JS ゼロで軽量、必要箇所だけ client JS | Next.js（SSR 不要でオーバースペック）/ 素の HTML（データ駆動にできない） |
| データ管理 | JSON + Zod スキーマ検証 | コード変更なしでゲーム・動画を追加。ビルド時に不正データを検出 | ヘッドレス CMS（無料枠でも運用が重い・依存が増える） |
| ホスティング | GitHub Pages | 完全無料・独自アカウント不要（GitHub のみ）・Actions で自動デプロイ | Cloudflare Pages（同等だがアカウント追加が必要）/ S3+CloudFront（有料） |
| CI/CD | GitHub Actions | push → ビルド → Pages デプロイを全自動化。無料枠内 | 手動デプロイ（運用が続かない） |

## 2. アーキテクチャ

```
[JSON データ] ──ビルド時──> [Astro SSG] ──HTML/CSS/最小JS──> [GitHub Pages]
 site/src/data/*.json         型検証(Zod)                      無料配信
                                                                 ▲
[git push] ──> [GitHub Actions] ── build & deploy ──────────────┘
```

- 完全静的サイト。サーバーサイド処理なし
- 進行度フィルタのみクライアント JS（`<script>` 直書きの Vanilla TS、フレームワーク不使用）

## 3. データモデル

`site/src/data/games/<game-id>.json` — 1 ゲーム 1 ファイル。

```jsonc
{
  "id": "sample-quest",            // URL スラッグ
  "title": "サンプルクエスト",
  "officialChannel": "https://www.youtube.com/@example",
  "checkpoints": [                  // 進行度の定義（順序 = 進行順）
    { "id": "ch1", "label": "第1章クリアまで" },
    { "id": "ch2", "label": "第2章クリアまで" },
    { "id": "clear", "label": "クリア済み" }
  ],
  "videos": [
    {
      "youtubeId": "XXXXXXXXXXX",   // YouTube 動画 ID（11桁）
      "title": "ローンチトレーラー",
      "type": "PV",                 // PV | キャラ紹介 | インタビュー | その他
      "safeAfter": "ch1",           // このチェックポイント到達後なら安全
      "reason": "第1章の舞台紹介が中心。以降のネタバレなし"
    }
  ]
}
```

- **ネタバレ判定**: `safeAfter` のチェックポイント位置 ≦ 選択中の位置 → 表示。超える場合は「未解禁 n 件」とだけ表示
- サムネイルは `https://i.ytimg.com/vi/<youtubeId>/hqdefault.jpg` を利用（YouTube 提供の公式サムネイル URL）
- Zod スキーマ（`site/src/lib/schema.ts`）でビルド時検証: `safeAfter` が checkpoints に存在しない場合はビルド失敗

## 4. 画面設計

| 画面 | パス | 内容 |
|---|---|---|
| トップ | `/` | サイト説明 + ゲームカード一覧（タイトル・動画数） |
| ゲーム詳細 | `/games/<id>/` | 進行度セレクタ + 動画カードリスト（種別バッジ・おすすめ理由・YouTube リンク）+ 未解禁件数表示 |
| 404 | `/404` | トップへの導線 |

- 進行度セレクタ: `<select>`。変更で即フィルタ + localStorage 保存（キー: `progress:<gameId>`）
- 初期表示は「最初のチェックポイント」（= 最も安全側）。JS 無効環境でもネタバレしない
- レスポンシブ: カードグリッド（モバイル 1 列 / デスクトップ 2〜3 列）。ダークテーマ基調

## 5. ディレクトリ構成（site/）

```
site/
├── astro.config.mjs          # base パス（GitHub Pages 用）等
├── package.json
├── tsconfig.json
├── src/
│   ├── data/games/*.json     # ゲームデータ（運用で増やすのはここだけ）
│   ├── lib/schema.ts         # Zod スキーマ + データローダー
│   ├── components/           # GameCard / VideoCard / ProgressSelector
│   ├── layouts/Base.astro    # 共通レイアウト
│   └── pages/
│       ├── index.astro
│       ├── 404.astro
│       └── games/[id].astro  # 動的ルート（getStaticPaths で全ゲーム生成）
└── public/                   # favicon 等
```

## 6. インフラ・デプロイ設計

- リポジトリ: GitHub（public）。`personal-site/` 直下を リポジトリルートとする
- デプロイ: `.github/workflows/deploy.yml` — main への push で `site/` をビルドし GitHub Pages へ
- URL: `https://<ユーザー名>.github.io/<リポジトリ名>/`（`astro.config.mjs` の `base` で対応）
- シークレット不要（Pages 公式 Action のみ。OIDC トークンで完結）

## 7. テスト方針

- ビルド時: Zod によるデータ整合性検証（型・参照整合）
- `npm run build` 成功 = 全ページ生成成功をもって静的サイトの健全性とする
- 手動確認: 進行度変更で未解禁動画が隠れること（ネタバレ防止の核）
