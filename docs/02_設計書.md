# 設計書 — 崩壊：スターレイル 開拓クエスト 動画ガイド

作成日: 2026-07-18 / 最終更新: 2026-07-19 / 作成: Claude（AI 駆動開発）

## 1. 技術選定

| 領域 | 採用 | 理由 |
|---|---|---|
| 言語 / FW | TypeScript / Astro（SSG） | コンテンツ主体・デフォルト JS ゼロ。動的挙動はチップ選択のみ |
| データ | JSON + Zod 検証 | コード変更なしで動画追加。ビルド時にネタバレ規則を機械検証 |
| ホスティング | GitHub Pages + Actions | 完全無料・自動デプロイ（初期構築から変更なし） |
| 調査 | 手動 Web 閲覧 + wiki + YouTube oEmbed | 規約準拠（yt-dlp / Data API 不使用）。ID 実在・公式チャンネル・タイトルを oEmbed で機械検証 |

## 2. 分類モデル（ネタバレ防止の核）

バージョン切替日マスタ（`research/data/versions-master.json`、全29バージョン・2ソース突合済み）を基準に:

- **before（来る前に見る）**: 公開日 ∈ [start(N-1), start(N)) の動画 → Ver.N ページ。キャラPV・予告番組が実装直前に公開される公式慣例に一致
- **after（本編クリア後推奨）**: 公開日 ∈ [start(N), start(N+1)) の動画のうち以下 → Ver.N ページの入れ子タブ
  - ユーザー指定リスト（ストーリー核心の紀行PV・EP 等）
  - **バージョン後半ピックアップキャラの PV**（自動判定: キャラPVが期間中盤に公開＝次バージョン開始まで8日以上）。次バージョン開始7日以内の公開は「次バージョン実装キャラの前日公開」とみなし before
- 例外は raw データの `phase` / `pinVersion`（+`movedFrom`/`note`）で個別に固定できる

**Zod によるビルド時検証**: before は公開日 < 掲載バージョン開始日 / after は当該期間内 / movedFrom 時の note 必須 / 参照整合 / id とファイル名の一致。違反はビルド失敗。

## 3. データモデル

```
site/src/data/
├── majors.json            # メジャーバージョン → 編名（例 "2": "ピノコニー"）
└── versions/<id>.json     # 1バージョン1ファイル（1.0 〜 4.4 の29ファイル）
```

```jsonc
// versions/2.0.json
{
  "id": "2.0",
  "name": "真夜中に夢を見るなら",     // 公式バージョン名
  "startDate": "2024-02-06",
  "videos": [
    {
      "youtubeId": "XXXXXXXXXXX",   // 11文字。oEmbed で実在・公式ch検証済みのもののみ
      "title": "…",
      "type": "trailblaze",         // trailblaze | character | version | music
      "publishedAt": "2024-02-01",
      "phase": "after",             // 省略時 before
      "reason": "…",                // 任意（UI 非表示・調査記録）
      "embeddable": false,          // 埋め込み不可動画はリンクのみ表示（省略時 true）
      "movedFrom": "1.6", "note": "…" // 機械分類から手動移動した場合の記録
    }
  ]
}
```

- 期間の終端・メジャーグループは `loadVersions()` / `groupByMajor()`（`site/src/lib/schema.ts`）が導出
- 種別: trailblaze=紀行PV・ストーリー（即興芝居・黄金の叙事詩PV含む） / character=キャラクターPV / version=バージョン予告・特別番組 / music=EP・MV・アニメーション

## 4. 画面設計

| 画面 | パス | 内容 |
|---|---|---|
| トップ | `/` | メジャーバージョンカード（編名・路線モチーフ・件数）＋「種別からまとめて見る（ネタバレ注意）」リンク |
| バージョンページ | `/v/1/`〜`/v/4/` | マイナーバージョンの横並びチップ（路線図・sticky）→ 選択で該当バージョンのみプルダウン展開。中は種別セクション＋「本編クリア後推奨の動画」入れ子折りたたみ。前後ナビ |
| 種別ページ | `/type/{character,trailblaze,music}/` | メジャーバージョンのトグル（初期展開）のみで区切った公開日順一気見 |
| 404 | `/404` | 一覧への導線 |

- 動画カード: 公式埋め込みプレイヤー（youtube-nocookie / lazy）・種別バッジ・公開日・YouTube リンク。サムネ直リンクは使用しない（規約対応）
- 全折りたたみ初期閉（バージョンページ）＝ JS 無効でもネタバレしない
- デザイン: 深宇宙紺 `#0b0e19` × 真鍮ゴールド `#d9a95c` の1アクセント。データ表記は等幅（時刻表モチーフ）。チップ列は「線路上の駅」を表すシグネチャ

## 5. ディレクトリ構成


## 6. 調査パイプライン

1. **収集**: 公式チャンネル・公式プレイリスト・wiki（wikiwiki「ゲーム外の公式コンテンツ」等）を通常閲覧の範囲で確認し `raw-videos.json` へ追記
2. **検証**: `verify-ids.mjs` が全 youtubeId を YouTube oEmbed に照会（実在・チャンネル名「崩壊：スターレイル」・正式タイトル）
3. **分類**: `classify.mjs` が §2 の規則で before/after × バージョンを機械判定（境界±3日はフラグ）
4. **レビュー**: `gen-review-md.mjs` → `review/videos-by-version.md` を人がレビュー、修正は raw に反映して再実行
5. **反映**: `gen-site-json.mjs` → `site/src/data/versions/*.json` を生成
