# 運用手順 — ゲーム進行度別 公式動画ガイド

作成日: 2026-07-18 / 作成: Claude（AI 駆動開発）

## 1. 初回公開手順（1 回だけ）

1. GitHub で **public リポジトリ**を作成する（例: `game-video-guide`）
2. このディレクトリ（`personal-site/`）をリポジトリルートとして push する
   ```bash
   git remote add origin https://github.com/<ユーザー名>/game-video-guide.git
   git push -u origin main
   ```
3. GitHub リポジトリの **Settings → Pages → Build and deployment → Source** を **GitHub Actions** に変更する
4. push 済みなら Actions の「Deploy to GitHub Pages」が自動実行される。完了後、
   `https://<ユーザー名>.github.io/game-video-guide/` で公開される

※ base パスは Actions 上で `GITHUB_REPOSITORY` から自動導出されるため、リポジトリ名を変えても設定変更は不要。

## 2. 日常運用: ゲーム・動画の追加

**コード変更は不要。JSON を編集して push するだけ。**

1. `site/src/data/games/<ゲームid>.json` を作成（既存の `sample-quest.json` をコピーして編集）
2. 記入ルール:
   - `id`: 英小文字ケバブケース（URL になる）
   - `checkpoints`: **進行順に**並べる（この順序がネタバレ判定の基準）
   - `videos[].youtubeId`: YouTube URL の `v=` 以降の 11 文字
   - `videos[].safeAfter`: 「このチェックポイントまで進んでいれば安全」な checkpoint の id
   - `videos[].type`: `PV` / `キャラ紹介` / `インタビュー` / `その他` のいずれか
3. ローカル確認: `cd site && npm run build`（データ不正はここでエラーになる）→ `npm run dev` で表示確認
4. **ネタバレ確認（必須）**: ゲームページで進行度を最初に戻し、先の動画が「未解禁 n 件」表示になることを確認
5. `git add` → commit → push（main への push で自動デプロイ）

サンプルデータ（`sample-quest.json`）は実データを 1 件登録したら削除してよい。

## 3. 保守

| 作業 | 頻度 | 内容 |
|---|---|---|
| 依存更新 | 半年に 1 回程度 | `cd site && npm update` → `npm run build` が通れば push |
| デプロイ失敗時 | 随時 | GitHub Actions のログを確認。大半はデータ不正（エラーメッセージに JSON のパスが出る） |
| リンク切れ確認 | 随時 | 公式が動画を削除するとサムネイルが出なくなる（サイトはプレースホルダー表示で壊れない）。該当エントリを JSON から削除 |

## 4. コスト

- GitHub（public リポジトリ + Actions + Pages）: **0 円**
- 外部サービス契約なし・シークレット登録なし
- 帯域: GitHub Pages のソフトリミット（100GB/月）は個人サイトでは実質無制限

## 5. 既知の制約

- 未解禁動画は CSS 非表示であり、HTML ソースを開けばタイトルは見える（「うっかりネタバレ」防止が目的で、意図的な閲覧までは防がない）
- 進行度はブラウザごとの保存（localStorage）。端末を変えると再選択が必要
- YouTube の動画自体の公開状態はコントロールできない（削除・非公開化されたら JSON から外す）
