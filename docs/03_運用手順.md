# 運用手順 — 崩壊：スターレイル 開拓クエスト 動画ガイド

作成日: 2026-07-18 / 最終更新: 2026-07-19 / 作成: Claude（AI 駆動開発）

## 1. 初回公開手順（1 回だけ）

1. GitHub で **public リポジトリ**を作成する（例: `hsr-trailblaze-video-guide`）
2. このディレクトリ（`personal-site/`）をリポジトリルートとして push する
   ```bash
   git remote add origin https://github.com/<ユーザー名>/<リポジトリ名>.git
   git push -u origin main
   ```
3. リポジトリの **Settings → Pages → Build and deployment → Source** を **GitHub Actions** に変更する
4. Actions の「Deploy to GitHub Pages」完了後、`https://<ユーザー名>.github.io/<リポジトリ名>/` で公開される

※ base パスは Actions 上で自動導出されるため、リポジトリ名の変更に設定変更は不要。

## 2. 日常運用A: 新バージョン公開時（6週間ごと）

1. **バージョンマスタ更新**: `research/data/versions-master.json` に新バージョン（id・名称・開始日）を追記。日付は公式プレスと攻略 wiki の2ソースで確認する
2. **新規動画の収集**: 公式チャンネル（@Houkaistarrail_jp）の新着から対象4種別（紀行PV・キャラPV・予告/バージョンPV・EP/MV/アニメ）を `research/data/raw-videos.json` に追記
   - youtubeId は動画 URL から転記（推測禁止）。生放送・Shorts・声優挨拶・システム紹介・グッズ告知は除外リストへ
   - ストーリー核心の動画・バージョン後半ピックアップキャラの PV は `"phase": "after"` を明示（キャラPVは期間中盤公開なら自動で after になる）
3. **パイプライン実行**（リポジトリルートで）:
   ```bash
   node research/scripts/classify.mjs        # 分類（境界±3日のフラグに注意）
   node research/scripts/verify-ids.mjs      # oEmbed で実在・公式ch検証
   node research/scripts/gen-review-md.mjs   # レビュー用MD生成 → 目視確認
   node research/scripts/gen-site-json.mjs   # site/src/data/versions/*.json へ反映
   ```
4. **新メジャーバージョン（X.0）の場合**: `site/src/data/majors.json` に編名を1行追加
5. **確認**: `cd site && npm run build` → `npm run dev` で表示確認。**ネタバレ確認（必須・§4）**を実施
6. `git add` → commit → push（main への push で自動デプロイ）

## 3. 日常運用B: 動画1本だけ追加・修正するとき

`research/data/raw-videos.json` に追記（または修正）→ §2-3 のパイプライン4コマンドを実行 → build 確認 → push。
site/src/data を直接編集した場合は次回パイプライン実行で上書きされるため、**必ず raw 側を正とする**こと。

## 4. ネタバレ確認（データ変更時は必須）

1. `cd site && npm run build` が通ること（Zod が「before=開始日前公開のみ / after=当該期間内のみ」等を機械検証）
2. `grep -o '<details[^>]*open' site/dist/v/*/index.html` が**何も出力しない**こと（バージョンページの折りたたみが静的に開いていない）
3. `grep -r 'i.ytimg.com' site/dist` が**何も出力しない**こと（サムネ直リンク禁止の維持）
4. ブラウザでバージョンページを開き、選んだバージョン以外が閉じていること・「本編クリア後推奨」が閉じていることを目視確認

## 5. 保守

| 作業 | 頻度 | 内容 |
|---|---|---|
| 依存更新 | 半年に1回程度 | `cd site && npm update` → build が通れば push |
| デプロイ失敗時 | 随時 | Actions ログを確認。大半はデータ不正（エラーに JSON パスと理由が出る） |
| 削除・非公開動画 | 随時 | `node research/scripts/verify-ids.mjs` を実行すると検出できる。該当を raw から除外リストへ移し再生成 |
| 埋め込み不可動画 | 発見時 | raw の該当動画に `"embeddable": false` → リンクのみ表示になる |

## 6. コスト・規約

- GitHub（public + Actions + Pages）: **0 円**。外部サービス契約・シークレットなし
- 規約対応の不変条件（変更禁止）: 公式チャンネル動画のみ / 埋め込みプレイヤーのみ（サムネ直リンク禁止） / ロゴ・公式アート不使用 / 非公式表記と権利帰属のフッター維持 / 広告なし / 調査に自動化ツール不使用

## 7. 既知の制約

- 「本編クリア後推奨」等の伏せは折りたたみ（HTML `<details>`）であり、意図的に開けば見える（うっかり防止が目的）
- キャラPV の公開日の一部は概算（`raw-videos.json` の `approx: true`）。境界±3日は classify がフラグを出すので注意
- YouTube 側で動画が削除・非公開化された場合は埋め込みがエラー表示になる（サイトは壊れない）。§5 の手順で除外する
